Create a new file at `src/app/api/qr/[id]/route.ts` with three exported handlers: GET, DELETE, and PUT.

All three handlers share these steps:
1. Auth gate: `getServerSession(authOptions)`, return 401 if no session.
2. Await `params` (Next.js 15 async params): `const { id } = await params`.

GET handler:
- Fetch the full record: `prisma.qrCode.findUnique({ where: { id }, include: { content: true, design: true } })`.
- If not found, return 404.
- Ownership check: if `record.userId !== session.user.id`, return 403.
- Return `NextResponse.json(record, { status: 200 })`.

```typescript
export async function GET(
  request: Request,
  { params }: { params: Promise<{ id: string }> }
) {
  const session = await getServerSession(authOptions);
  if (!session?.user?.id) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });
  }
  const { id } = await params;
  const record = await prisma.qrCode.findUnique({
    where: { id },
    include: { content: true, design: true },
  });
  if (!record) {
    return NextResponse.json({ error: 'Not found' }, { status: 404 });
  }
  if (record.userId !== session.user.id) {
    return NextResponse.json({ error: 'Forbidden' }, { status: 403 });
  }
  return NextResponse.json(record, { status: 200 });
}
```

DELETE handler (ownership check via findUnique with select):
- `prisma.qrCode.findUnique({ where: { id }, select: { userId: true } })`. If not found, return 404. If `qrCode.userId !== session.user.id`, return 403.
- `await prisma.qrCode.delete({ where: { id } })` — Prisma cascades delete QrContent and QrDesign automatically (onDelete: Cascade in schema).
- Return `NextResponse.json({ success: true }, { status: 200 })`.

PUT handler (same ownership check pattern as DELETE):
